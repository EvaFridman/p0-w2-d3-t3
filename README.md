# p0-w2-d3-t3 · Прилипающая шапка и кнопка «наверх»

> Тип: синтез (каркас grid/flex + позиционирование) · Репозиторий: один отдельный

## Задача

Взять одну из своих страниц с готовым каркасом (портфолио `d5`, или раскладку на grid из
`d2-t3`) и добавить два элемента поверх: шапку, которая прилипает при прокрутке, и кнопку
«наверх», висящую в углу окна. Главное — не сломать существующую раскладку и доступность.

```
твоя страница (каркас grid/flex)
┌──────────────────────────────────────────────────┐
│ sticky-шапка — прилипает к верху при скролле     │
│                                                  │
│   длинный контент, прокрутка...                  │
│                                                  │
│                              fixed-кнопка [ ↑ ]  │
│                              висит в углу окна   │
└──────────────────────────────────────────────────┘
раскладка не сломана, доступность та же
```

## Что нужно сделать

Скопируй проект в новый репозиторий. Сделай шапку `sticky`, добавь `fixed`-кнопку «наверх»
(обычная ссылка-якорь на верх страницы — без JS). Каркас grid/flex остаётся как был, навигация
с клавиатуры тоже.

## Что нужно получить

- шапка прилипает к верху при прокрутке, контент под ней не подпрыгивает и не прячется;
- кнопка «наверх» зафиксирована в углу окна (`fixed`), видна при прокрутке, ведёт на верх
  страницы по якорю;
- кнопка не перекрывает важный контент и доступна с клавиатуры (Tab + видимый фокус);
- существующая раскладка и порядок чтения не изменились.

## Критерии приёмки

* Шапка `sticky`, кнопка `fixed`; обе ведут себя корректно при прокрутке.
* Каркас и доступность не сломаны: страница проходится с клавиатуры, фокус виден.
* Кнопка «наверх» работает без JS (якорная ссылка) и не перекрывает контент под собой.
